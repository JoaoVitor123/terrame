customError2 

Stop the simulation with an error.

Arguments

  • #1: A string describing the error message.

Usage

customError2("error message")

customWarning2 

Print a warning. If TerraME is executing in the debug mode, it stops the simulation with an error.

Arguments

  • #1: A string describing the warning.

Usage

customWarning2("warning message")

defaultTableValue2 

Verify the default value of a given attribute of a named table. It adds the attribute with the default value in the table if it does not exist, stops with an error (incompatibleTypeMsg2()) if the value has a different type, or shows a warning (defaultValueWarning2()) if it is equal to the default value.

Arguments

  • #1: A named table.
  • #2: A string with the name of the attribute.
  • #3: The default value (any type).

Usage

t = {x = 5}
defaultTableValue2(t, "y", 8)

defaultValueMsg2 

Return a message indicating that the modeler is using the default value for a given argument and therefore it could be removed.

Arguments

  • #1: A string with the name of the argument.
  • #2: A number or string or boolean value with the default value for the argument.

Usage

defaultValueMsg2("dbtype", "mysql")

defaultValueWarning2 

Show a strict warning if the attribute of a table has the default value. If TerraME is running in the debug mode, the simulation stops with an error. The warning message comes from defaultValueMsg2().

Arguments

  • #1: A string with the attribute name.
  • #2: The default value.

Usage

defaultValueWarning2("size", 2)

deprecatedFunction2 

Show an error indicating a deprecated function. The error comes from deprecatedFunctionMsg2().

Arguments

  • #1: A string with the name of the deprecated function.
  • #2: A string indicating hot to proceed to replace the deprecated function.

Usage

deprecatedFunction2("abc", "def")

deprecatedFunctionMsg2 

Return a message indicating that a function is deprecated and must be replaced.

Arguments

  • #1: A string with the name of the deprecated function.
  • #2: A string indicating hot to proceed to replace the deprecated function.

Usage

deprecatedFunctionMsg2("abc", "def")

incompatibleTypeError2 

Stop the simulation with an error of a wrong type for an argument of a function. The error message is the return value of incompatibleTypeMsg2().

Arguments

  • #1: A string with an attribute name, or a numeric position of the argument in a function with non-named arguments.
  • #2: A string with the possible type (or types).
  • #3: The value passed as argument with wrong type.

Usage

incompatibleTypeError2("cell", "Cell", Agent{})

incompatibleTypeMsg2 

Return an error message for incompatible types.

Arguments

  • #1: A string with an attribute name, or a numeric position of the argument in a function with non-named arguments.
  • #2: A string with the possible type (or types).
  • #3: The value passed as argument with wrong type.

Usage

incompatibleTypeMsg2("dbType", "string", 2)

incompatibleValueError2 

Stop the simulation with an error of a wrong value for an argument of a function. The error message comes from incompatibleValueMsg2().

Arguments

  • #1: A string with an attribute name (for functions with named arguments), or position (otherwise).
  • #2: A string with the expected values for the argument.
  • #3: The wrong value passed as argument.

Usage

incompatibleValueError2("position", "one of {1, 2, 3}", 4)

incompatibleValueMsg2 

Return an error message for incompatible values.

Arguments

  • #1: A string with an attribute name (for functions with named arguments), or position (otherwise).
  • #2: A string with the expected values for the argument.
  • #3: The wrong value passed as argument.

Usage

incompatibleValueMsg2("dbType", "one of {1, 3, 4}", 2)

integerArgument2 

Verify whether a given argument of a non-named function is integer. The error message comes from integerArgumentMsg2().

Arguments

  • #1: A number with the position of the argument in the function.
  • #2: The valued used as argument to the function call.

Usage

integerArgument2(1, 2.3)

integerArgumentMsg2 

Return a message indicating that a given argument of a function should be integer.

Arguments

  • #1: The name of the argument. It can be a string or a number.
  • #2: The valued used as argument to the function call.

Usage

integerArgumentMsg("target", 7.4)
integerArgumentMsg2(2, 5.1)

integerTableArgument2 

Verify whether a given argument of a named function is integer. The error message comes from integerArgumentMsg2().

Arguments

  • #1: A named table.
  • #2: A string with the attribute name.

Usage

mtable = {bbb = 2.3}
integerTableArgument2(table, "bbb")

Return a message indicating that a given file extension is incompatible.

Arguments

  • #1: A string with an attribute name (for functions with named arguments), or position (otherwise).
  • #2: A string with the incompatible file extension.

Usage

mandatoryArgument2 

Verify whether a given argument of a function with non-named arguments belongs to the correct type. The error message comes from mandatoryArgumentMsg2() and incompatibleTypeMsg2().

Arguments

  • #1: A number wiht the position of the argument in the function.
  • #2: A string with the required type for the argument.
  • #3: The value used as argument to the function call.

Usage

mandatoryArgumentError2 

Stop the simulation with an error indicating that a given argument is mandatory. The error message comes from mandatoryArgumentMsg2().

Arguments

  • #1: The name of the argument (a string) or the position of the argument in the function (a number).

Usage

mandatoryArgumentError("target")
mandatoryArgumentError2(2)

mandatoryArgumentMsg2 

Return a message indicating that a given argument of a function is mandatory.

Arguments

  • #1: The name of the argument. It can be a string or a number.

Usage

mandatoryArgumentMsg("target")
mandatoryArgumentMsg2(2)

mandatoryTableArgument2 

Verify whether a named table contains a mandatory argument. It stops with an error if the value is nil or if it does not belong to the required type. The error message comes from mandatoryArgumentMsg2() or incompatibleTypeMsg2().

Arguments

  • #1: A named table.
  • #2: A string with the attribute name.
  • #3: A string with the required type for the attribute.

Usage

mtable = {bbb = 3, ccc = "aaa"}
mandatoryTableArgument2(mtable, "bbb", "string")

namedArgumentsMsg2 

Return a message indicating that the arguments of a function must be named.

Usage

namedArgumentsMsg2()

optionalArgument2 

Verify whether an optional argument of a function with non-named arguments belongs to the correct type. The error message comes from incompatibleTypeMsg2(), only if the argument is not nil.

Arguments

  • #1: A number with the position of the argument in the function.
  • #2: A string with the required type for the argument.
  • #3: The value used as argument to the function call.

Usage

optionalArgument2(1, "string", argument)

optionalTableArgument2 

Verify whether a named table contains an optional argument. It stops with an error if the value is not nil and it has a type different from the required type. The error comes from incompatibleTypeError2().

Arguments

  • #1: A named table.
  • #2: A string with the argument name.
  • #3: A string with the required type for the attribute.

Usage

mtable = {bbb = 3, ccc = "aaa"}
optionalTableArgument2(mtable, "bbb", "string")

positiveArgument2 

Verify whether a given argument of a function with non-named arguments is positive. The error message comes from positiveArgumentMsg2().

Arguments

  • #1: A number with the position of the argument in the function.
  • #2: The value used as argument to the function call.
  • #3: A boolean value indicating whether zero should be included (the default value is false).

Usage

positiveArgument2(1, -2)

positiveArgumentMsg2 

Return a message indicating that a given argument of a function should be positive.

Arguments

  • #1: The name of the argument. It can be a string or a number.
  • #2: The value used as argument to the function call.
  • #3: A boolean value indicating whether zero should be included (the default value is false).

Usage

positiveArgumentMsg("target", -5)
positiveArgumentMsg2(2, -2, true)

positiveTableArgument2

Arguments

  • #1: A named table.
  • #2: A string with the name of the argument.
  • #3: A boolean value indicating whether zero should be included (the default value is false).

Usage

mtable = {bbb = -3}
positiveTableArgument2(table, "bbb")

resourceNotFoundMsg2

strictWarning2 

Print a strict warning. This warning is shown only in the strict mode. If TerraME is executing in the debug mode, it stops the simulation with an error.

Arguments

  • #1: A string describing the warning.

Usage

strictWarning2("warning message")

suggestion2 

Return a suggestion for a wrong string value. The suggestion must have a Levenshtein's distance of less than 60% the size of the string, otherwise it returns nil.

Arguments

  • #1: A string.
  • #2: A table with string indexes with the possible suggestions.

Usage

t = {
    blue = true,
    red = true,
    green = true
}

suggestion2("gren", t) -- "green"

switchInvalidArgument2 

Stop the simulation with an error because the user did not choose a correct option. This function supposes that there is a set of available options described as string idexes of a table so it tries to find an approximate string to be shown as a suggestion. Otherwise, it shows all the available options. The error messages come from switchInvalidArgumentSuggestionMsg2() and switchInvalidArgumentMsg2().

Arguments

  • #1: A string with the attribute name.
  • #2: A string with wrong value passed as argument.
  • #3: A table with string indexes describing the available options.

Usage

t = {
    blue = true,
    red = true,
    green = true
}

switchInvalidArgument2("attribute", "gren", t)

switchInvalidArgumentMsg2 

Return a message for a wrong argument value showing the options.

Arguments

  • #1: A string with the value of the attribute.
  • #2: A string with the name of the attribute.
  • #3: A table whose indexes indicate the available options.

Usage

local options = {
    aaa = true,
    bbb = true,
    ccc = true
}
switchInvalidArgumentMsg2("ddd", "attr", options)

switchInvalidArgumentSuggestionMsg2 

Return a message for a wrong argument value showing the most similar option.

Arguments

  • #1: A string with the value of the attribute.
  • #2: A string with the name of the attribute.
  • #3: A string with a suggestion to replace the wrong value.

Usage

switchInvalidArgumentSuggestionMsg2("aab", "attr", "aaa")

tableArgumentMsg2 

Return a message indicating that the argument of a function must be a table.

Usage

tableArgumentMsg2()

toLabel2 

Return a string with a literal description of a parameter name. It is useful to work with init() when the model will be available through a graphical interface. When using graphical interfaces, it converts upper case characters into space and lower case characters and convert the first character of the string to uppercase. Otherwise, it returns the name of the parameter itself.

Arguments

  • #1: A string with the parameter name.
  • #2: A string with the name of the table the parameter belongs to. This parameter is optional.

Usage

toLabel2("maxValue") --  'Max Value' (with graphical interface) or 'maxValue' (without)

unnecessaryArgumentMsg2 

Return a message indicating that a given argument is unnecessary.

Arguments

  • #1: A string or number or boolean value.
  • #2: A possible suggestion for the argument. This parameter is optional.

Usage

unnecessaryArgumentMsg("file")
unnecessaryArgumentMsg2("filf", "file")

valueNotFoundError2 

Stop the simulation with an error due to a wrong value for an argument. The error message comes from valueNotFoundMsg2().

Arguments

  • #1: A string with an attribute name (for functions with named arguments), or position (otherwise).
  • #2: The valued used as argument to the function call.

Usage

valueNotFoundError2(1, "neighborhood")

valueNotFoundMsg2 

Return a message indicating that a given argument of a function is mandatory.

Arguments

  • #1: A string with an attribute name (for functions with named arguments), or position (otherwise).
  • #2: The valued used as argument to the function call.

Usage

verify2 

Verify a given condition, otherwise it stops the simulation with an error.

Arguments

  • #1: A value of any type. If it is false or nil, the function generates an error.
  • #2: A string with the error to be displayed.

Usage

verify2(2 < 3, "wrong operator")

verifyNamedTable2 

Verify if a given object is a named table. It generates errors if it is nil, if it is not a table, or if it has numeric indexes. The error messages come from tableArgumentMsg2() and namedArgumentsMsg2().

Arguments

  • #1: A value of any type.

Usage

t = {value = 2}
verifyNamedTable2(t)

verifyUnnecessaryArguments2 

Verify whether the user has passed only the allowed arguments for a function, showing a strict warning otherwise. The warning comes from unnecessaryArgumentMsg2(). This function returns the number of unnecessary arguments found.

Arguments

  • #1: A named table with the arguments used in the function call. The indexes of this table will be verified.
  • #2: A non-named table with the allowed arguments.

Usage

t = {value = 2}
verifyUnnecessaryArguments2(t, {"target", "select"})
